# UI 优化评审

## 目标

- 先解决会伤害用户信任和完成度的界面问题。
- 默认优先做高价值、小范围、低风险的 UI 优化。
- 任何会改变页面结构、导航路径、主入口优先级、核心交互方式的改动，先确认再做。

## 当前结论

整体视觉方向已经有基础风格，不是“不能看”，但当前最大问题不是配色或圆角，而是：

- 首页展示的信息和真实数据存在割裂。
- 部分页面仍暴露原型态、演示态、开发态文案。
- 中英文混用较多，产品完成度被明显拉低。
- 首页、添加好友、设置页的信息架构还不够产品化。

## 优先级

### PUI1 信任修复

目标：先把用户一眼就会质疑的点处理掉。

- 首页消息卡片改为真实消息摘要，不再显示写死人物和写死文案。
  涉及文件：`bruh/Views/Home/HomeScreen.swift`
- 朋友圈页移除“写死展示内容”“固定写入本地表”这类原型暴露文案，改成用户可理解的正式表达。
  涉及文件：`bruh/Views/Feed/FeedView.swift`
- 消息列表补空状态和搜索空状态，避免页面直接空白。
  涉及文件：`bruh/Views/Messages/MessagesScreen.swift`

说明：

- 这一轮可以不改导航结构。
- 这一轮可以尽量不改视觉骨架。
- 这一轮收益最高，风险最低。

### PUI2 文案统一

目标：把产品语言统一成一套用户可感知的正式表达。

- 联系人表单里的英文表单项改成中文。
  涉及文件：`bruh/Views/Contacts/ContactsView.swift`
- 相册页里的 `Today`、`Yesterday`、`No Album Yet`、`photos` 等文案统一。
  涉及文件：`bruh/Views/Album/AlbumView.swift`
- 设置页里的 `Display Mode`、`Use HomeScreen Mode` 统一成中文产品语气。
  涉及文件：`bruh/Views/Settings/SettingsScreen.swift`
- 消息页里的 `Failed to send`、`my bruh · online/offline` 等表达统一。
  涉及文件：`bruh/Views/Messages/MessagesScreen.swift`

说明：

- 这一轮属于低风险清理。
- 能显著提升完成度，但不应单独晚于 PUI1。

### PUI3 首页信息架构

目标：让首页更像产品首页，而不是概念化桌面。

- 降低大时钟的视觉权重。
- 减少首页重复导航入口。
- 提高“消息 / 动态 / 联系人”这三个核心任务的首屏可读性。
- 决定首页定位：
  - 方案 A：桌面式 launcher
  - 方案 B：消息驱动 dashboard

涉及文件：

- `bruh/Views/Home/HomeScreen.swift`
- `bruh/ContentView.swift`

说明：

- 这一轮会触及信息架构，必须先确认。
- 这是中等风险改动，不建议在 PUI1 之前做。

### PUI4 添加鸽们页转化优化

目标：让“添加鸽们”从静态名人目录，升级为可解释、可转化的推荐入口。

- 增加“为什么推荐给你”的解释层。
- 让“热门推荐”和“人气榜”更像真实推荐，而不是静态堆卡片。
- 强化 CTA 后的反馈，让“等待中”之外的信息更完整。

涉及文件：

- `bruh/Views/Contacts/AddBruhView.swift`

说明：

- 这一轮可能会调整卡片层级和信息密度，需要先确认。

### PUI5 设置页补完

目标：让设置页和主产品视觉语言一致，并具备基础产品层级。

- 从系统默认 `List` 风格升级为品牌化设置页。
- 明确账号、偏好、通知、实验开关、关于等分组。

涉及文件：

- `bruh/Views/Settings/SettingsScreen.swift`

说明：

- 这一轮不急，但做完会让产品完成度提升明显。

## 暂不建议立即做的改动

- 大规模换色或重做视觉主题。
- 大幅改动 tab / home 模式切换逻辑。
- 一次性重构所有页面布局。
- 在没有统一文案策略前做大批量装饰性 UI 调整。

## 建议执行顺序

1. `PUI1 信任修复`
2. `PUI2 文案统一`
3. `PUI3 首页信息架构`
4. `PUI4 添加鸽们页转化优化`
5. `PUI5 设置页补完`

## 准备先做的首轮改动

如果你点头，首轮我建议只做下面这些，控制范围，不大拆页面：

1. 首页消息卡片切到真实数据，保留现有卡片样式，不先改整体布局。
2. 朋友圈页移除开发态/演示态文案，保留现有页面结构。
3. 消息列表补空状态和搜索空状态。
4. 顺手统一一批最刺眼的英文文案：
   - 消息发送失败提示
   - 相册页日期和空状态
   - 设置页开关文案
   - 联系人表单文案

## 需你确认后再做

- 首页是否要从“桌面式”往“dashboard 式”调整。
- 是否允许我修改首页首屏组件的视觉主次。
- 是否允许我调整“添加鸽们”页面的信息层级和卡片结构。
- 是否允许我重做设置页视觉，而不仅是改文案。
